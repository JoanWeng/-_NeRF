# NeRF 程式碼解析

## 專案概述

本專案從零實作 NeRF (Neural Radiance Fields)，輸入多張照片，透過 COLMAP
估算相機姿態，訓練一個 MLP 神經網路學習 3D 場景的體積密度與顏色，最後能從新視角渲染畫面。

---

## 整體流程

```
照片集 → COLMAP (SfM 相機姿態估算) → transforms.json
                                              ↓
         訓練：對每條射線取樣 3D 點 → NeRF MLP → 體積渲染 → MSE Loss
                                              ↓
         測試：用未看過的相機姿態渲染新視角 → PSNR / SSIM 評估
```

這個流程的核心概念是：要從 2D 照片還原 3D 場景，你必須知道每張照片是從哪裡拍的（相機姿態），以及場景中每個位置的顏色與不透明度（NeRF 學的東西）。這兩件事分別對應到 COLMAP 和 NeRF 訓練。

## 2. COLMAP 整合（相機姿態估算）

### 2.1 `run_cmd(cmd)`

```python
def run_cmd(cmd):
    is_bat = str(cmd[0]).endswith(".bat")
    r = subprocess.run(
        cmd, capture_output=True,
        encoding=sys.stdout.encoding or "utf-8", errors="replace",
        shell=is_bat,
    )
    if r.returncode != 0:
        raise RuntimeError(f"COLMAP failed at '{cmd[1]}':\n{r.stderr[-400:]}")
```

這是一個包裝函式，用來執行 COLMAP 的外部指令。Windows 上 `.bat` 批次檔需要 `shell=True` 才能執行，一般 `.exe` 則不需要。若指令失敗，會截取 stderr 的最後 400 字元來顯示錯誤訊息。

### 2.2 `quat_to_rot(qw, qx, qy, qz)`

```python
def quat_to_rot(qw, qx, qy, qz):
    return np.array([
        [1-2*(qy**2+qz**2),   2*(qx*qy-qz*qw),   2*(qx*qz+qy*qw)],
        [  2*(qx*qy+qz*qw), 1-2*(qx**2+qz**2),   2*(qy*qz-qx*qw)],
        [  2*(qx*qz-qy*qw),   2*(qy*qz+qx*qw), 1-2*(qx**2+qy**2)],
    ])
```

**什麼是四元數 (quaternion)？**
三維空間中的旋轉可以用 3×3 矩陣描述，也可以用 4 個數字 `(w, x, y, z)` 組成的四元數描述。兩者可以互相轉換。COLMAP 回傳的相機旋轉是用四元數表示的，所以需要轉回旋轉矩陣才能進行座標變換。公式來自四元數到旋轉矩陣的標準轉換：

```
R = [1-2(y²+z²)    2(xy-zw)     2(xz+yw) ]
    [ 2(xy+zw)    1-2(x²+z²)    2(yz-xw) ]
    [ 2(xz-yw)     2(yz+xw)    1-2(x²+y²)]
```

### 2.3 `colmap_txt_to_transforms(txt_dir, image_dir, out_path)`


這段程式將 COLMAP 的純文字輸出轉為 NeRF 標準格式 `transforms.json`。

**相機到世界矩陣 (camera-to-world, c2w) 是怎麼來的？**

COLMAP 的 `images.txt` 中每張照片記錄了：
- 四元數 `(qw, qx, qy, qz)` → 表示相機在世界座標中的旋轉
- 平移向量 `(tx, ty, tz)` → 表示相機在世界座標中的位置

但 COLMAP 儲存的是 **world-to-camera (w2c)** 矩陣，代表「世界點 → 到相機座標」的變換。而 NeRF 需要的是反過來的 **camera-to-world (c2w)**。

轉換方式：
```python
R = quat_to_rot(qw, qx, qy, qz)    # 旋轉矩陣
t = np.array([tx, ty, tz])          # 平移向量
c2w = np.eye(4)
c2w[:3, :3] = R.T                   # w2c 的旋轉轉置 = c2w 的旋轉
c2w[:3,  3] = -R.T @ t              # w2c 的平移反推回世界座標
c2w[:, 1]  *= -1                    # 翻轉 Y 軸（COLMAP 上→NeRF 下）
c2w[:, 2]  *= -1                    # 翻轉 Z 軸（COLMAP 前→NeRF 後）
```

**為什麼要翻轉 Y、Z？**
COLMAP 的座標系慣例與 NeRF 論文不同：
- COLMAP：Y 向上，Z 向後
- NeRF：Z 向上，Y 向後
所以需要轉換。

**為什麼焦距要用 camera_angle_x？**
`camera_angle_x = 2 × arctan(W / (2 × focal))` 是水平視角。NeRF 的 `transforms.json` 標準格式使用視角而非焦距，因為焦距會隨圖片縮放改變，但視角是固定的相機屬性。

### 2.4 `run_colmap(image_dir, output_dir)`

```python
def run_colmap(image_dir, output_dir):
    ws     = output_dir / "colmap_ws"
    db     = ws / "database.db"
    sparse = ws / "sparse"
    ws.mkdir(parents=True, exist_ok=True)
    sparse.mkdir(exist_ok=True)
    c      = str(COLMAP_BIN)

    for cmd in [
        [c, "feature_extractor",
         "--database_path", str(db),
         "--image_path",    str(image_dir),
         "--ImageReader.single_camera",       "1",
         "--SiftExtraction.max_num_features", "16384"],
        [c, "exhaustive_matcher",
         "--database_path", str(db),
         "--TwoViewGeometry.min_num_inliers", "10"],
        [c, "mapper",
         "--database_path", str(db),
         "--image_path",    str(image_dir),
         "--output_path",   str(sparse)],
    ]:
        print(f"  → {cmd[1]} ...")
        run_cmd(cmd)
```

這是一整個 SfM（Structure from Motion）管線，做了三件事：

#### 步驟 1：特徵提取 (feature_extractor)

對每張照片提取 **SIFT (Scale-Invariant Feature Transform)** 特徵點。SIFT 能找到照片中的關鍵點（如角點、邊緣、紋理變化處），並對每個關鍵點產生一個 128 維的描述向量。這些描述向量具有**尺度不變性**和**旋轉不變性**，也就是說同一個點從不同角度、遠近拍照，特徵描述仍然相似。

參數 `--ImageReader.single_camera 1`：告知 COLMAP 所有照片都是用同一台相機拍的，共享相同的內參（焦距、感光元件等）。
參數 `--SiftExtraction.max_num_features 16384`：每張照片最多提取 16384 個特徵點，提高匹配成功率。

#### 步驟 2：特徵匹配 (exhaustive_matcher)

將所有照片兩兩配對，比對它們的 SIFT 特徵描述向量。若兩張照片中兩個特徵的描述向量相似度高，則視為一組**匹配對**。

`--TwoViewGeometry.min_num_inliers 10`：若兩張照片之間的匹配少於 10 個點，則視為無效配對。

#### 步驟 3：地圖建構 (mapper)

這是 SfM 的核心。它用匹配對進行三角測量（triangulation），同時估計：
- **每張照片的相機姿態**（位置與旋轉）
- **匹配點的 3D 座標**（這就是前面提到的稀疏點雲）

演算法流程：
1. 從匹配最多的兩張照片開始（初始化）
2. 估算它們的相對姿態，三角化出初始 3D 點
3. 逐步加入新照片（registration），用已有的 3D 點估算新照片姿態
4. 加入新 3D 點（triangulation），然後對所有點進行 bundle adjustment（同時優化相機姿態和 3D 點位置，最小化重投影誤差）

#### 步驟 4：模型轉換 (model_converter)

將 COLMAP 二進位輸出（`.bin`）轉為可讀的純文字格式（`.txt`），方便 Python 解析。

#### 快取機制

```python
cache = COLMAP_OUT / "transforms.json"
if cache.exists():
    # 跳過 COLMAP
else:
    # 執行 COLMAP
```

COLMAP 執行一次約需數分鐘，若已經跑過，`transforms.json` 就會存在，可直接跳過。

## 3. NeRF 模型架構

```python
class NeRF(nn.Module):
    def __init__(self, pos_dim, dir_dim, hidden=256):
        super().__init__()
        self.block1 = nn.Sequential(
            nn.Linear(pos_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden),  nn.ReLU(),
            nn.Linear(hidden, hidden),  nn.ReLU(),
            nn.Linear(hidden, hidden),  nn.ReLU(),
        )
        self.block2 = nn.Sequential(
            nn.Linear(hidden + pos_dim, hidden), nn.ReLU(),
            nn.Linear(hidden, hidden),           nn.ReLU(),
            nn.Linear(hidden, hidden),           nn.ReLU(),
            nn.Linear(hidden, hidden + 1),
        )
        self.color_head = nn.Sequential(
            nn.Linear(hidden + dir_dim, hidden // 2), nn.ReLU(),
            nn.Linear(hidden // 2, 3),                nn.Sigmoid(),
        )

    def forward(self, pos_enc, dir_enc):
        h     = self.block1(pos_enc)
        h     = self.block2(torch.cat([h, pos_enc], dim=-1))
        sigma = torch.relu(h[..., 0])
        rgb   = self.color_head(torch.cat([h[..., 1:], dir_enc], dim=-1))
        return rgb, sigma
```

### 網路結構圖

```
輸入位置編碼 pos_enc (63 維)
    ↓
  block1: Linear(63→256) → ReLU
          Linear(256→256) → ReLU
          Linear(256→256) → ReLU
          Linear(256→256) → ReLU
    ↓ (得到 256 維特徵)
  ─────────────────────────────────────────────
  將 block1 輸出與原始 pos_enc 拼接 (256+63=319 維)
  ─────────────────────────────────────────────
    ↓
  block2: Linear(319→256) → ReLU
          Linear(256→256)  → ReLU
          Linear(256→256)  → ReLU
          Linear(256→257)  ← 輸出 257 維
    ↓
  拆開為兩部分：
    ├── sigma = ReLU(h[..., 0])        → 體積密度 (1 維純量)
    └── features = h[..., 1:]          → 顏色特徵 (256 維)
    ↓ (將 features 與方向編碼 dir_enc 拼接: 256+27=283 維)
  color_head: Linear(283→128) → ReLU
              Linear(128→3)   → Sigmoid
    ↓
  rgb = (R, G, B)    → 每個值在 [0,1] 之間
```

### 為什麼要這樣設計？

**問題：為什麼位置編碼要到 63 維？為什麼不直接用 3D 座標 (x, y, z)？**

NeRF 論文發現：直接輸入 (x, y, z) 訓出來的網路只能學到低頻（模糊）的場景幾何，無法重現高頻細節（如紋理、邊緣）。這是因為神經網路傾向於學習低頻函數（spectral bias / frequency bias 現象）。

解決方法：把 3D 座標用不同頻率的 sin/cos 展開成高維向量，如同把一個簡單訊號用傅立葉級數展開，讓網路更容易學到高頻成分。

**為什麼要有 skip connection（從 block1 接到 block2）？**

一般的深度網路在反向傳播時，梯度在層層傳遞中容易消失（vanishing gradient）。skip connection 讓梯度有一條捷徑可以直接流到前面層，同時也讓位置資訊不會被深層網路稀釋。

**為什麼 sigma 和 rgb 分開？**

- `sigma`（體積密度）只與**位置**有關。同一個 3D 點從哪個方向看，密度都一樣。
- `rgb`（顏色）與**位置和方向都有關**。這是因為真實世界中，物體表面會有視角相關的反射效果（如金屬光澤、鏡面反射）。

如果顏色不考慮方向，NeRF 只能學到 diffuse（漫射）顏色，看起來會像沒有反光的塑膠物體。

## 4. 位置編碼 (Positional Encoding)

```python
def positional_encoding(x, num_freqs):
    freqs = 2.0 ** torch.linspace(0, num_freqs - 1, num_freqs, device=x.device)
    return torch.cat([x] + [fn(f * x) for f in freqs for fn in (torch.sin, torch.cos)], dim=-1)
```

### 具體做了什麼？

假設 `x = (0.5, 1.2, -0.8)`，`num_freqs = 2`：

```
freqs = [2^0, 2^1] = [1.0, 2.0]
```

對每個頻率，對每個維度做 sin 和 cos：

```python
# 頻率 1.0:
sin(1.0 * 0.5), cos(1.0 * 0.5),   # x 維度
sin(1.0 * 1.2), cos(1.0 * 1.2),   # y 維度
sin(1.0 * -0.8), cos(1.0 * -0.8), # z 維度

# 頻率 2.0:
sin(2.0 * 0.5), cos(2.0 * 0.5),
sin(2.0 * 1.2), cos(2.0 * 1.2),
sin(2.0 * -0.8), cos(2.0 * -0.8),
```

最終輸出：原始 3 維 + sin/cos 各 2 頻率 × 3 維度 × 2 函數 = `3 + 12 = 15` 維

在程式中 `POS_FREQS=10`，所以位置編碼輸出維度是 `3 + 3 × 2 × 10 = 63`。

### 數學直觀

把位置編碼想像成 GPS 座標的表示方式。光說「經度 121.5 度」不夠精確，但如果你也說「121.5 度的 sin 是 0.38，cos 是 0.92」，再搭配更高頻的「242 度的 sin 是 -0.88，cos 是 0.47」……越多頻率，越能精確定義一個位置。

## 5. 射線生成 (`get_rays`)

```python
def get_rays(H, W, focal, c2w):
    i, j = torch.meshgrid(
        torch.arange(W, dtype=torch.float32, device=c2w.device),
        torch.arange(H, dtype=torch.float32, device=c2w.device),
        indexing="xy",
    )
    dirs   = torch.stack([(i-W*0.5)/focal, -(j-H*0.5)/focal, -torch.ones_like(i)], dim=-1)
    rays_d = (dirs[..., None, :] * c2w[:3, :3]).sum(-1)
    rays_o = c2w[:3, 3].expand(rays_d.shape)
    return rays_o, rays_d
```

這一步將「照片中的像素位置」轉換為「從相機射出的 3D 射線」。

### 逐步拆解

**Step 1：像素網格**
```python
i, j = meshgrid(0..W-1, 0..H-1)
```
`i` 是 x 方向的像素索引（0 到 W-1），`j` 是 y 方向的像素索引（0 到 H-1）。

**Step 2：相機座標系中的射線方向**
```python
dirs = [(i - W/2) / focal, -(j - H/2) / focal, -1]
```
- `(i - W/2) / focal`：將像素座標轉為相機感光元件上的實際位置。除以 `focal` 是因為針孔相機模型中，3D 點在感光元件上的位置 = `focal × (x/z)`，這裡我們設 `z = -1`（朝向鏡頭前方）。
- `-(j - H/2) / focal`：負號是因為影像的 y 軸向下，但相機座標系的 y 軸向上。
- `-1`：固定 z = -1，表示射線方向指向相機前方。

**Step 3：轉到世界座標**
```python
rays_d = (dirs[..., None, :] * c2w[:3, :3]).sum(-1)
```
相機座標系中的方向 × 旋轉矩陣 = 世界座標系中的方向。

**Step 4：射線原點**
```python
rays_o = c2w[:3, 3].expand(rays_d.shape)
```
相機中心就是 `c2w` 矩陣的平移部分，所有射線都從同一點出發。

### 範例

如果相機在 (0, 0, 0) 面向 -Z，focal=100，那麼像素 (200, 150) 產生的射線就是：
- 方向：((200-400)/100, -(150-300)/100, -1) = (-2, 1.5, -1)
- 原點：(0, 0, 0)

## 6. 射線取樣 (`sample_points`)

```python
def sample_points(rays_o, rays_d, near, far, n_samples, perturb=False):
    t_vals = torch.linspace(0, 1, n_samples, device=rays_o.device)
    z_vals = (near*(1-t_vals) + far*t_vals).expand(rays_o.shape[0], n_samples)
    if perturb:
        mids   = 0.5 * (z_vals[..., 1:] + z_vals[..., :-1])
        upper  = torch.cat([mids, z_vals[..., -1:]], dim=-1)
        lower  = torch.cat([z_vals[..., :1], mids],  dim=-1)
        z_vals = lower + (upper - lower) * torch.rand_like(z_vals)
    return rays_o[..., None, :] + rays_d[..., None, :] * z_vals[..., :, None], z_vals
```

### 為什麼需要取樣？

NeRF 需要知道 3D 空間每一點的密度和顏色，但我們不能把所有 3D 點都輸入網路（無限多點）。所以沿著每條射線「離散地」取 64 個點來近似。

### 訓練模式 vs 推論模式

- **推論時 `perturb=False`**：在 `[near, far]` 區間內均勻取 64 個點，z 值固定。這樣渲染結果穩定，不會閃爍。
- **訓練時 `perturb=True`**：先在 `[near, far]` 間分 64 個區間，然後每個區間內隨機取一個點。這樣做的好處是：
  1. 每個訓練步的取樣位置不同 → 網路學到的是**連續空間**的資訊，而不是過擬合離散點位置
  2. 隨機性相當於正則化，幫助網路泛化

### 擾動取樣的數學

若無擾動：`z_vals = [0, 1/63, 2/63, ..., 1]` 線性映射到 `[near, far]`
有擾動時，對每個區間 `[z_k, z_{k+1}]`，在區間內均勻隨機取一個值。

## 7. 體積渲染 (`volume_render`)

```python
def volume_render(rgb, sigma, z_vals, bg_color=1.0):
    deltas  = torch.cat([z_vals[..., 1:] - z_vals[..., :-1],
                         torch.full_like(z_vals[..., :1], 1e10)], dim=-1)
    alpha   = 1.0 - torch.exp(-sigma * deltas)
    T       = torch.cumprod(torch.cat([torch.ones_like(alpha[..., :1]),
                                       1-alpha+1e-10], dim=-1), dim=-1)[..., :-1]
    weights = T * alpha
    rgb_map = (weights[..., None] * rgb).sum(-2) + (1 - weights.sum(-1, keepdim=True)) * bg_color
    return rgb_map, weights
```

### 核心公式

體積渲染是 NeRF 的核心。對於一條射線，最終像素顏色 = 所有 3D 取樣點顏色的加權平均，權重由該點的密度和位置決定。

#### 變數解釋

- `sigma`：體積密度，越大表示該點越不透明（實心物體）
- `deltas`：相鄰取樣點的間距，乘以 sigma 決定了該區間的總不透明度
- `alpha`：該區間的不透明度，範圍 [0, 1]。`alpha = 1 - exp(-sigma × delta)`
  - sigma 很大（物體）→ alpha 接近 1 → 光線被擋住
  - sigma 很小（空氣）→ alpha 接近 0 → 光線穿透

#### 逐步計算

**Step 1：區間距離**
```
z_vals = [z0, z1, z2, ..., z62]
deltas = [z1-z0, z2-z1, ..., z62-z61, 1e10]
```
最後一個區間設為 1e10（極大），確保射線末端的光線全被吸收。

**Step 2：不透明度**
```
alpha_k = 1 - exp(-sigma_k * delta_k)
```
物理意義：光線經過第 k 個區間時，被吸收的比例。

**Step 3：累積穿透率**
```
T_k = product(1 - alpha_j) for j < k
```
物理意義：光線成功到達第 k 個區間（沒被前面吸收）的機率。

**Step 4：權重**
```
weight_k = T_k * alpha_k
```
物理意義：第 k 個區間對最終像素顏色的貢獻比例。

**Step 5：顏色合成**
```
rgb_map = sum(weight_k * rgb_k) + (1 - sum(weights)) * bg_color
```
- 第一項：所有 3D 點貢獻的顏色
- 第二項：背景顏色（未被任何物體擋住的光線穿透到背景）

### 為什麼用 alpha = 1 - exp(-sigma × delta)？

這是光線穿過參與介質（participating media）時的衰減公式。假設介質密度均勻，光線穿過距離 `delta` 後，強度衰減到原來的 `exp(-sigma × delta)`，所以被吸收的比例就是 `1 - exp(-sigma × delta)`。

### 與傳統電腦圖學的關係

這個公式本質上就是 **alpha compositing**（alpha 混合）的連續版本，用在體積渲染、煙霧、雲、毛髮等半透明材質。傳統的 alpha 混合是對圖層使用：
```
result = front_rgb * front_alpha + back_rgb * (1 - front_alpha)
```
NeRF 把這個概念延伸到沿射線的連續密度場。

## 8. 完整渲染一張圖片 (`render_pose`)

```python
def render_pose(model, pose, H, W, focal):
    model.eval()
    rays_o, rays_d = get_rays(H, W, focal, pose.to(device))
    rays_o, rays_d = rays_o.reshape(-1, 3), rays_d.reshape(-1, 3)
    chunks = []
    with torch.no_grad():
        for k in range(0, rays_o.shape[0], BATCH_RAY):
            ro, rd  = rays_o[k:k+BATCH_RAY], rays_d[k:k+BATCH_RAY]
            pts, zv = sample_points(ro, rd, NEAR, FAR, N_SAMPLES)
            pe      = positional_encoding(pts.reshape(-1, 3), POS_FREQS)
            de      = positional_encoding(rd[:, None].expand_as(pts).reshape(-1, 3), DIR_FREQS)
            rgb_raw, sigma_raw = model(pe, de)
            rgb_map, _ = volume_render(rgb_raw.reshape(-1, N_SAMPLES, 3),
                                       sigma_raw.reshape(-1, N_SAMPLES), zv)
            chunks.append(rgb_map.cpu())
    return torch.cat(chunks).reshape(H, W, 3).numpy().clip(0, 1)
```

### 為什麼要分塊 (chunking)？

一張 1734×2312 的圖片有約 400 萬條射線，每條 64 個點 → 2.56 億個 3D 點。不可能一次全部送入 GPU 記憶體。

所以每 1024 條射線為一組（`BATCH_RAY`），分批次渲染，最後拼接回完整圖片。

### 為什麼要 `model.eval()` 和 `with torch.no_grad()`？

- `model.eval()`：關閉 dropout、batch norm 等訓練行為
- `torch.no_grad()`：不計算梯度，節省 GPU 記憶體並加速推論（因為不需要儲存中間變數用於反向傳播）

## 9. NEAR / FAR 自動估算 (`estimate_near_far`)

```python
def estimate_near_far(txt_dir, poses, percentile=5):
    pts3d_path = txt_dir / "points3D.txt"
    pts = []
    with open(pts3d_path, encoding="utf-8") as f:
        for line in f:
            if line.startswith("#") or not line.strip():
                continue
            parts = line.split()
            pts.append([float(parts[1]), float(parts[2]), float(parts[3])])
    # ... 對每個相機位置，計算到所有 3D 點的距離
    # ... 取百分位數估算 near/far
```

### 為什麼要自動估算？

NEAR 和 FAR 定義了沿射線取樣的範圍，是體積渲染中至關重要的超參數：
- NEAR 太小 → 浪費計算資源在相機附近的空區域
- NEAR 太大 → 可能切掉靠近相機的物體
- FAR 太小 → 可能切掉場景後方
- FAR 太大 → 浪費計算資源在空區域

### 具體做法

1. **讀取稀疏點雲**：`points3D.txt` 包含 2862 個 3D 點（來自 COLMAP 的 SfM）
2. **對每個相機位置**：計算到所有 3D 點的歐幾里得距離
3. **收集所有距離**：得到幾萬個（2862 點 × 30 相機）深度值
4. **取百分位數**：
   - `near = percentile(depths, 5%) × 0.8`：取第 5 百分位，再打折確保不會漏掉東西
   - `far  = percentile(depths, 95%) × 1.2`：取第 95 百分位，再擴張確保涵蓋完整

### 若點雲為空（fallback）

如果 `points3D.txt` 不存在（COLMAP 重建失敗），改用相機位置分布來估算：
```
cam_origins = 所有相機位置
radius      = 相機位置到平均中心的距離最大值 + 0.5
near        = max(radius × 0.1, 0.05)
far         = radius × 3.0
```

```python
def load_dataset(test_ratio=0.2):
    meta   = json.loads((COLMAP_OUT / "transforms.json").read_text(encoding="utf-8"))
    frames = meta["frames"]
    n_test = max(1, int(len(frames) * test_ratio))
    # ... (依此類推)
    train_imgs, train_poses = load_frames(frames[:-n_test])
    test_imgs,  test_poses  = load_frames(frames[-n_test:])
    H, W  = train_imgs[0].shape[:2]
    focal = 0.5 * int(W/SCALE) / np.tan(0.5 * meta["camera_angle_x"]) * SCALE
    # ... (估算 near/far)
```

### split 方式

30 張照片：
- 訓練：前 24 張
- 測試：後 6 張

這是最簡單的 split 方式。確保測試照片沒被看過，才能評估模型的新視角合成能力。

### 圖片預處理流程

```
讀取圖片（imageio）
    ↓
歸一化到 [0, 1]（除以 255）
    ↓
處理 RGBA 或灰階格式（轉為 RGB）
    ↓
縮小到原始大小的 SCALE=0.5 倍（PIL.LANCZOS 高品質縮放）
    ↓
轉為 PyTorch tensor
```

### 焦距計算

`transforms.json` 中儲存的是 `camera_angle_x`（水平視角），但模型需要的是 `focal`（像素焦距），所以需要換算回來：

```python
focal = 0.5 * int(W/SCALE) / np.tan(0.5 * meta["camera_angle_x"]) * SCALE
```

其中 `W` 是原始圖片寬度，`SCALE=0.5` 因為圖片已經縮小一半。

為什麼有 `0.5 × int(W/SCALE)`？因為 `camera_angle_x = 2 × arctan(W / (2 × focal))`，反推得到 `focal = W / (2 × tan(camera_angle_x / 2))`。由於圖片寬度原本是 `W` 但已經縮小為 `W × SCALE`，在 `focal` 也等比例縮小時，實際上要用原始 `W` 計算再乘 `SCALE`。

## 11. 訓練迴圈

```python
np.random.seed(42); torch.manual_seed(42)
model     = NeRF(pos_dim, dir_dim).to(device)
optimizer = torch.optim.Adam(model.parameters(), lr=5e-4)
scheduler = torch.optim.lr_scheduler.ExponentialLR(optimizer, gamma=0.1**(1/N_ITERS))
losses, t0 = [], time.time()

for _ in tqdm(range(N_ITERS), desc="Training"):
    idx             = np.random.randint(n_train)
    img, pose       = train_imgs[idx].to(device), train_poses[idx].to(device)
    rays_o, rays_d  = get_rays(H, W, focal, pose)
    rays_o, rays_d  = rays_o.reshape(-1, 3), rays_d.reshape(-1, 3)
    target          = img.reshape(-1, 3)
    sel             = np.random.choice(rays_o.shape[0], BATCH_RAY, replace=False)
    rays_o, rays_d, target = rays_o[sel], rays_d[sel], target[sel]
    pts, z_vals     = sample_points(rays_o, rays_d, NEAR, FAR, N_SAMPLES, perturb=True)
    pe              = positional_encoding(pts.reshape(-1, 3), POS_FREQS)
    de              = positional_encoding(rays_d[:, None].expand_as(pts).reshape(-1, 3), DIR_FREQS)
    rgb_raw, sigma_raw = model(pe, de)
    rgb_map, _      = volume_render(rgb_raw.reshape(BATCH_RAY, N_SAMPLES, 3),
                                    sigma_raw.reshape(BATCH_RAY, N_SAMPLES), z_vals)
    loss = ((rgb_map - target)**2).mean()
    optimizer.zero_grad(); loss.backward(); optimizer.step(); scheduler.step()
    losses.append(loss.item())

torch.save(model.state_dict(), "ckpt_real.pt")
```

### 訓練演算法（逐次解讀）

```
每步迭代：
  1. 隨機選一張訓練照片                → 避免每次看同一張
  2. 生成該照片所有射線 (H×W 條)       → 從相機姿態計算
  3. 從所有射線中隨機選 1024 條         → 降低計算量
  4. 沿每條射線取 64 個 3D 點（擾動取樣）→ 離散化連續空間
  5. 對所有 3D 點做位置編碼和方向編碼    → 幫助網路學高頻
  6. NeRF 前向計算 → 得到 (rgb, sigma)  → 網路預測
  7. 體積渲染 → 得到預測像素顏色         → compositing
  8. MSE Loss（預測 vs 真實像素）        → 誤差計算
  9. 反向傳播更新網路權重                → 梯度下降
  10. 學習率指數衰減                     → 收斂穩定
```

### 為什麼一次只用一張圖的一部分射線？

理論上，理想訓練應該用所有照片的所有射線，但這計算量太大。NeRF 用隨機取樣策略（stochastic sampling）：
- 每次只選一張照片中的 1024 條射線
- 15000 步 × 1024 = 1536 萬條射線
- 資料集中有 24 張 × 400 萬條 ≈ 9600 萬條射線
- 每步只看到約 0.01% 的射線，但因為每次都是隨機且數量夠大，網路仍然能收斂

### 學習率排程

```python
scheduler = ExponentialLR(gamma=0.1 ** (1 / N_ITERS))
```

這是一個指數衰減排程，公式：
```
lr_at_step_t = initial_lr × (0.1)^(t / N_ITERS)
```

- t=0：lr = 5e-4
- t=7500（一半）：lr ≈ 5e-4 × 0.1^(0.5) ≈ 1.58e-4
- t=15000（最終）：lr = 5e-5

初期大學習率快速靠近最優解，後期小學習率微調。

### 為什麼訓練要 111 分鐘？

每步計算量：
- 1024 條射線 × 64 點 = 65536 個 3D 點
- 每個點 63 維位置編碼 + 27 維方向編碼 → MLP 前向
- 反向傳播計算梯度 → 更新 1.2M 個參數
- 15000 步 → 約 9.8 億次點評估

```python
model.eval()
psnrs, ssims, renders = [], [], []

print(f"PSNR: {mean_psnr:.2f} dB  |  SSIM: {mean_ssim:.4f}")
```

### PSNR（峰值信噪比）

```
PSNR = 10 × log10(MAX² / MSE)
     = 10 × log10(1² / MSE)      （因為像素值在 [0,1]）
```

- MAX = 像素最大值（這裡是 1.0）
- MSE = 預測與真實圖像的均方誤差
- PSNR 以 dB 為單位，越高越好
- 22.80 dB 的 PSNR 表示 MSE ≈ 0.0052，這個結果對 NeRF 來說屬於中等水準

### SSIM（結構相似性）

SSIM 不只比較像素值，還比較亮度、對比度、結構三個層面：

```
SSIM(x, y) = [l(x,y)]^α × [c(x,y)]^β × [s(x,y)]^γ
```

- `l`：亮度比較（均值）
- `c`：對比度比較（標準差）
- `s`：結構比較（相關係數）
- 範圍 [−1, 1]，1 表示完全相同。0.8517 表示還原度不錯。

### 為什麼 PSNR 22.80 dB 不算非常高？

一般 NeRF 論文中，LLFF 資料集的 PSNR 可達 26-27 dB。這裡偏低的原因：
1. 照片數量較少（24 張訓練）
2. `N_ITERS=15000` 相對較少（論文中通常訓練 20 萬步）
3. 手持拍攝可能有小幅度對不準
4. `SCALE=0.5` 損失了高頻細節

| 訓練 | MSE + Adam + LR decay | 最小化渲染與真實圖像差異 | 每步 1024 射線 × 64 點，約 111 分鐘 |
| 評估 | PSNR / SSIM | 量化渲染品質 | PSNR 22.80 dB, SSIM 0.8517 |
